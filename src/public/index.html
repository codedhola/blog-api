API Endpoints

CRUD for Blog

  • GET => api/v1/blogs

    • get all blogs

  • GET => api/v1/blogs/:id

    • Get a blog, This routes also gets the author details of the blog

  • POST => api/v1/blogs

    • Create a new Blog, This route is protected!, only admin users can create a blog. This route accepts data's from users

    Which are :=> { title, description and body }

  • PATCH => api/v1/blogs/:id

    • Edit an existing Blog by providing it ID as a parameter and edit it data { title, description or body }

  • DELETE => api/v1/blogs/:id

    • Delete a blog in this Endpoint by just providing it ID

User Authentication

  • POST => api/v1/users/auth/sign-up

    • 'firstName', 'lastName', 'email', 'gender' and 'password' must be provided in order to sign up as a user

  • POST => api/v1/users/auth/login

    • 'Email' and 'password' While Signing up as a user must be used correctly in order to be logged in

Features

  • User Authentication => Sign up & Login
  • Protected Routes
  • Admin Create blog, edit and delete blog
  • Read both published and draft blogs
  • Can search blog by title and tags using pagination
